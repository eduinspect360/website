Design decisions — mobile ROIJ
Offline-first architecture
Inspection data, all lookup tables and the AI phrase bank are cached on first load (~300KB). The inspector works throughout the school day without connectivity. The amber sync bar shows pending entries at all times.
AI phrase bank (offline)
When a judgement grade is selected, ED AI immediately surfaces bullet points from the offline phrase bank appropriate to that grade and section. Accept adds the text in one tap — no server call required.
Section progress & conflict handling
Section tabs show Done (green) or pending state. Inspectors own specific sections — the system prevents two inspectors editing the same section simultaneously, eliminating conflict scenarios by design.
Security always visible
“This entry is logged” on every form makes the audit trail visible — a trust signal and a behavioural governance control. On reconnect, offline entries are submitted using the authenticated session token.

How the three-tier language system works
Tier 1 — UI chrome
.NET Resource strings serve all navigation labels, buttons, form fields and table headers in the selected language from the server. No client-side DOM manipulation. Secure and fast.
Tier 2 — ED AI responses
The Claude API system prompt includes the user's language preference from their server-side session. All chatbot responses, report summaries and data insights appear in the selected language automatically.
Tier 3 — Inspection narratives
Reports and observations are stored in the language they were written. ED AI can translate individual sections on request — preserving the integrity of the original professional judgement text.
